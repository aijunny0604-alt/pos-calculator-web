#### 프롬프트 전면 재설계
- **차종 약어 사전**: 벨N/아N/코나N 등 → 정식명 변환 테이블
- **카테고리별 매칭 규칙**: 머플러팁(카본/NPK/SNPK), 밴딩, 자바라, 다운파이프, 레듀샤, 플랜지, 클램프, 레조, 실리콘 호스 등 각각 제품명 형식과 키워드 매핑
- **신뢰도 판정 기준 3단계 상세화**:
  - HIGH: 코드 명시, 직경+각도+재질 모두 있음, 차종+부품 1~2개로 좁혀짐
  - MEDIUM: 부품 종류 명확하나 스펙 일부 누락
  - LOW: 부품 종류 자체가 모호
- **동의어/오타 변환표**: 스텐→스덴, 벤딩→밴딩, 레듀서→레듀샤 등
- **few-shot 예시 9개**: 제품코드, 자연어, 모호한 입력 등 다양한 패턴
- temperature: 0.2 → 0.1

#### JS 매칭 로직 강화 (findProduct 7단계)
1. 정확 일치
2. 대소문자 무시
3. 공백/하이픈 정규화 후 비교 (신규)
4. 부분 문자열 + 숫자 ±1 허용
5. 정규화된 부분 문자열 (신규)
6. 토큰 스코어링 - 키워드 겹침 점수 (신규)
7. tolerance 매칭 폴백

#### 테스트 케이스 (8회, 39개 항목)
- 약어/오타: "벨n 싱글", "스텐 밴딩", "scf 103 듀얼"
- 카톡 자연어: "사장님 주문합니다", "열개", "하나요", "보내주세요"
- 제품코드: "CFK 80", "SNPK 89", "NPK 100 T타입"
- 복합 분리: "카본듀얼이랑 싱글 각각 1개씩" → 2항목 분리
- 모호한 입력: "머플러 팁 하나", "밴딩파이프 좀"
- 희귀 제품: 실리콘 엘보, 진공호스, 스포츠촉매, 단엘보, 레조
- 실전 주문서: 9항목 전부 확실 매칭

## 4. AI 분석 결과 카드 UI 미니멀 리디자인 (커밋 `f3ceb49`)

### 문제
- 결과 카드에 테두리가 과다: 외곽 border + 원본텍스트 strip border-b + 수량 컨트롤 border + 대안 버튼 border + alternatives border-t

### 수정
- **카드 외곽**: border → box-shadow (선택된 것만 primary outline)
- **원본 텍스트**: 배경 분리 strip 제거 → 카드 내 상단 텍스트로 통합
- **신뢰도 뱃지**: 텍스트 뱃지 → 컬러 dot (초록/노랑/빨강)
- **체크박스**: border-2 → 배경색만 있는 미니멀 체크
- **수량 컨트롤**: border 있는 box → borderless (배경색만)
- **편집/삭제 버튼**: 배경 있는 버튼 → ghost 버튼 (hover 시만 배경)
- **대안 후보 칩**: border-t 구분선 + border 버튼 → 구분선 없이 연한 배경 칩
- **카드 간격**: space-y-2 → space-y-1.5
