## Plan (계획)

### 문제 정의
- NPK/SNPK 머플러팁 제품의 색상 코드(S=실버, T=티탄블루, B=블랙) 매칭이 부정확
- "티탄블루90 듀얼팁 좌우" 입력 시 `듀얼 NPK 80D - S` (실버)로 잘못 매칭됨
- 기대 결과: `듀얼 NPK 89D - T` (티탄블루)

### 근본 원인 분석
1. **프롬프트에 색상코드 설명 부재**: `-S`를 "스퀘어(스타일)"로 잘못 설명, 실제로는 "실버(색상)"
2. **색상 키워드 → 색상코드 변환 규칙 없음**: "티탄블루", "블랙", "은색" 등의 키워드와 T/B/S 코드 매핑이 전무
3. **few-shot 예시 편향**: 9개 예시 모두 `-S`(실버)만 사용, 색상 지정 예시 0개
4. **동의어 맵 미비**: "좌우"→듀얼, "파란색"→티탄블루, "검정"→블랙 등 매핑 없음
5. **사이즈 근사 매칭 규칙 없음**: 90mm → 89mm 변환 미지원

## Do (실행)

### 변경 사항 (TextAnalyze.jsx)

#### 1. 프롬프트 - 색상코드 체계 명확화
**이전**: `-S(스퀘어) 기본값으로 매칭` (색상 설명 없음)
**이후**:
- NPK/SNPK 제품명 구조 상세 설명 추가
- `D` = 듀얼, `S` = 싱글 (사이즈 뒤 = 팁 개수)
- `S` = 실버, `T` = 티탄블루, `B` = 블랙 (하이픈 뒤 = 색상코드)

#### 2. 색상 키워드 → 색상코드 변환표 추가
| 사용자 입력 | 색상코드 |
|------------|---------|
| 티탄, 티탄블루, titan, 타이탄, 파랑, 블루 | T |
| 실버, silver, 은색, 실바, 기본 | S |
| 블랙, black, 검정, 검은, 까만, 흑색 | B |

#### 3. few-shot 예시 대폭 확대 (9개 → 15개)
- **색상 지정 예시 6개 신규 추가**: 티탄블루, 블랙, 실버, 슬롯+색상 등
- 색상 미지정 시 medium confidence + alternatives 패턴 명시
- 기존 예시의 색상 미지정 NPK를 medium confidence로 수정

#### 4. 분석 규칙 3개 추가
- **규칙 9**: 색상 키워드 최우선 매칭 (무시 금지)
- **규칙 10**: "좌우" = 듀얼 매핑
- **규칙 11**: 사이즈 근사 매칭 (90→89, 115→114)

#### 5. JS 동의어 맵 확장
- 색상: 티탄블루/블루/파랑 → 티탄, 실바/은색 → 실버, 검정/까만/흑색 → 블랙
- 팁 타입: 좌우/양쪽/한쌍 → 듀얼

#### 6. 동의어/오타 변환표 확장
- 색상 키워드 매핑 추가
- "90 → 89mm" 사이즈 근사 규칙 추가
